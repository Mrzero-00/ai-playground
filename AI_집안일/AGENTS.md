# AI_집안일 에이전트 운영 규칙

## 제품 목표

가구 형태에 맞는 집안일을 추천하고 반복 일정, 알림, 완료 기록을 통해 사용자가 꾸준히 집을 관리하도록 돕는 앱인토스 미니앱을 만든다.

## 공통 규칙

- 모든 작업은 `docs/apps-in-toss-policy.md`를 필수 개발·출시 게이트로 적용한다.
- 공식 앱인토스 정책과 프로젝트 정책이 충돌하면 공식 정책을 우선한다.
- 정책 감사표에서 `출시 차단`으로 표시된 항목이 남아 있으면 출시 가능하다고 보고하지 않는다.
- 모바일 화면과 앱인토스 WebView 환경을 우선한다.
- 사용자를 압박하거나 죄책감을 주는 표현을 사용하지 않는다.
- 가구 형태, 성별, 혼인 여부에 대한 고정관념을 만들지 않는다.
- 추천 일정은 사용자가 변경하거나 끌 수 있어야 한다.
- 로그인 도입 전 데이터는 저장소 추상화 뒤의 로컬 저장소에 보관한다.
- 위생·안전 관련 AI 안내는 검증된 기준 없이 단정하지 않는다.
- 새 기능은 타입 검사와 프로덕션 빌드를 통과해야 한다.
- 외부 링크, 로그인, 결제, 광고, 생성형 AI, 기기 권한을 추가할 때는 구현 전에 정책 영향도를 문서화한다.
- 생성형 AI 결과를 노출하면 최초 고지, 결과 라벨, 안전 필터와 운영 대응 절차를 함께 구현한다.
- 커밋 메시지는 `AI_집안일 / 작업내용` 형식을 사용한다.
- 다른 프로젝트 및 루트의 관련 없는 파일은 수정하거나 커밋하지 않는다.

## 역할

### 제품·데이터 에이전트

- 요구사항, 사용자 흐름, 데이터 모델, 완료 조건을 관리한다.
- 모호한 정책은 가정과 확정 사항을 구분해 문서화한다.
- 담당 문서: `docs/product-concept.md`, `docs/mvp-spec.md`

### UI 에이전트

- 모바일 우선 화면, 접근성, 빈 상태와 오류 상태를 담당한다.
- 재사용 가능한 컴포넌트와 디자인 토큰을 우선한다.
- 담당 영역: `src/components`, `src/styles.css`

### 도메인 에이전트

- 집 프로필, 추천 규칙, 반복 일정, 기록 계산을 담당한다.
- UI나 특정 저장 기술에 의존하지 않는 순수 로직을 우선한다.
- 담당 영역: `src/domain`, `src/hooks`, `src/data`

### 앱인토스 연동 에이전트

- 앱인토스 SDK, 권한, 알림, 번들 및 출시 요건을 공식 문서로 검증한다.
- 플랫폼 기능이 준비되지 않으면 웹 대안과 후속 서버 작업을 구분한다.
- 담당 문서 및 영역: `docs/notification-plan.md`, `granite.config.ts`

### 통합·검증 에이전트

- `App.tsx`에서 기능을 통합하고 전체 사용자 흐름을 검증한다.
- 타입 검사, 빌드, 브라우저 시나리오 테스트 후 단계별로 커밋한다.
- 앱인토스 정책 감사표를 갱신하고 출시 차단 항목을 명시한다.

## 현재 MVP 범위

- 로그인 제외
- 집 프로필 설정
- 프로필 기반 기본 집안일 추천
- 커스텀 집안일과 반복 주기
- 오늘 해야 할 집안일과 완료 처리
- 집안일 활성화 및 비활성화
- 수행 히스토리
- 지원 가능한 범위의 알림 설정

## 범위 밖

- 서버 계정 및 기기간 동기화
- 가족 초대와 공동 분담
- 점수, 레벨, 소셜 공유
- 결제
- AI 채팅 및 생성형 추천
