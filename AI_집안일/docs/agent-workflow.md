# 에이전트 작업 구성

## 작업 분해 원칙

기능을 제품 정책, 화면, 도메인 로직, 플랫폼 연동, 통합 검증으로 나눈다. 각 작업자는 가능한 한 서로 다른 파일 영역을 담당하고, 통합 담당자가 최종 사용자 흐름과 충돌을 조정한다.

## MVP 작업 배분

| 역할 | 작업 | 결과물 |
| --- | --- | --- |
| 제품·데이터 | 요구사항, 집 프로필 항목, 추천 규칙, 완료 조건 | `docs/mvp-spec.md` |
| UI | 프로필, 홈, 집안일 관리, 커스텀 추가 화면 | `src/components/*`, `src/styles.css` |
| 도메인 | 반복 일정, 추천 목록, 로컬 저장, 완료 기록 | `src/domain/*`, `src/data/*`, `src/hooks/*` |
| 앱인토스 연동 | 공식 알림 지원 범위와 도입 계획 | `docs/notification-plan.md` |
| 통합·검증 | 상태 연결, 예외 처리, 빌드와 브라우저 테스트 | `src/App.tsx`, 검증 결과 |

## 단계별 완료 조건

### 1단계: 설계

- MVP 요구사항과 제외 범위가 문서에 명시된다.
- 프로필, 집안일, 반복 일정, 수행 기록 데이터가 정의된다.
- 알림의 현재 구현과 후속 서버 구현 범위가 구분된다.

### 2단계: 핵심 기능

- 첫 방문 사용자가 집 프로필을 저장할 수 있다.
- 프로필 조건에 맞는 추천 집안일이 자동 생성된다.
- 이름과 반복 주기를 입력해 커스텀 집안일을 추가할 수 있다.
- 오늘 할 일을 완료하면 다음 예정일과 히스토리가 갱신된다.

### 3단계: 품질 검증

- 새로고침 후에도 데이터가 유지된다.
- 빈 상태와 설정 전 상태가 자연스럽게 표시된다.
- 모바일 폭에서 가로 스크롤이나 잘림이 없다.
- 타입 검사, 웹 빌드, 앱인토스 번들 빌드가 성공한다.

## Git 운영

- 설계 및 에이전트 구성: `AI_집안일 / MVP 설계 및 에이전트 구성`
- 핵심 기능 구현: `AI_집안일 / 프로필 및 집안일 관리 구현`
- 검증과 알림 연동: `AI_집안일 / 알림 및 MVP 검증`

각 커밋은 `AI_집안일` 폴더만 포함하고 완료된 작업 단위마다 `main`에 푸시한다.
