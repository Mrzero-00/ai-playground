# AI 집안일 앱인토스 개발 정책

## 1. 목적

이 문서는 AI 집안일 미니앱의 기획, 디자인, 개발, 검수 및 운영에 적용하는 필수 기준이다. 앱인토스 공식 서비스 오픈 정책과 비게임 출시 가이드를 최소 기준으로 삼으며, 공식 정책이 변경되면 이 문서도 갱신한다.

공식 기준:

- [서비스 오픈 정책](https://developers-apps-in-toss.toss.im/intro/guide.html)
- [비게임 출시 가이드](https://developers-apps-in-toss.toss.im/checklist/app-nongame.html)
- [서비스별 주의사항](https://developers-apps-in-toss.toss.im/intro/caution.html)

## 2. 출시 차단 원칙

아래 항목 중 하나라도 충족하지 못하면 출시 가능한 상태로 표시하거나 출시 번들을 제출하지 않는다.

- 앱인토스에서 제한하는 서비스 또는 콘텐츠를 포함하지 않는다.
- 미니앱의 핵심 기능이 외부 앱 설치나 외부 웹사이트에 의존하지 않는다.
- 로그인은 토스 로그인만 사용한다.
- 결제 도입 시 실물 상품은 토스페이, 디지털 상품은 인앱결제만 사용한다.
- 광고 도입 시 앱인토스가 제공하는 광고 상품만 사용한다.
- 앱인토스 비게임 내비게이션과 뒤로가기 정책을 준수한다.
- 사용자 식별과 데이터 삭제·보존 정책이 검증된다.
- 앱 내 기능, 딥링크, 뒤로가기, 종료 흐름이 실제 토스 샌드박스에서 동작한다.
- 생성형 AI 결과를 제공하면 사전 고지와 결과물 표시를 적용한다.
- 개인정보 처리방침, 이용약관, 고객 문의 경로를 제공한다.

## 3. 서비스 및 콘텐츠

### 허용 범위

- 집 프로필 설정
- 맞춤 집안일 추천
- 반복 일정과 완료 기록
- 공동 집과 구성원 초대
- 집안일 담당자 지정과 구성원 통계
- 생활 관리 정보 및 루틴 제공

### 금지 및 제한

- 금융상품 광고·중개, 투자 조언, 사행성 기능을 추가하지 않는다.
- 의료 진단이나 치료로 해석될 수 있는 조언을 제공하지 않는다.
- 불법 행위나 위험 행동을 유도하는 청소·생활 정보를 제공하지 않는다.
- 동일 기능을 테마만 바꿔 별도 미니앱으로 반복 출시하지 않는다.
- 기존 서비스 홍보만을 목적으로 미니앱을 운영하지 않는다.

## 4. 완결형 사용자 경험

- 집 생성, 참여, 프로필 설정, 집안일 추가, 완료 및 기록 조회는 미니앱 안에서 끝나야 한다.
- 외부 앱 설치 문구, 앱마켓 링크, 설치 혜택을 제공하지 않는다.
- 법적 고지 등 허용되는 경우 외에는 외부 링크를 핵심 흐름에 사용하지 않는다.
- 공유 링크는 운영 환경에서 `intoss://` 스킴을 사용하고 `intoss-private://`를 사용자 공유에 사용하지 않는다.
- 앱 내 기능으로 등록한 항목은 해당 스킴에서 바로 접근하고 동일한 기능을 완결적으로 제공한다.

## 5. 로그인과 데이터

- 로그인 도입 전 로컬 데이터는 데모·개발 단계로만 취급한다.
- 운영 계정은 토스 로그인으로만 식별한다.
- 토스 로그인에서 받은 식별자는 파트너 서버에서 검증하고 임의 클라이언트 값으로 사용자를 신뢰하지 않는다.
- 사용자, 집, 구성원 소속, 집안일, 완료 기록을 분리해 저장한다.
- 토스 로그인 연결 해제 및 회원 탈퇴 시 관련 개인정보 삭제 절차를 제공한다.
- 공동 집에서 탈퇴한 사용자의 이름과 활동 기록 보존 범위는 개인정보 처리방침에 명시한다.
- 필요한 최소 정보만 수집하고 전송 및 저장 구간을 보호한다.
- 앱 종료와 재접속, 기기 변경 후에도 운영에 필요한 데이터가 일관되게 유지되어야 한다.

## 6. 내비게이션과 UI/UX

- 앱인토스 비게임 내비게이션 바를 사용한다.
- 토스 내비게이션의 뒤로가기와 자체 뒤로가기를 중복 노출하지 않는다.
- 최초 화면에서 뒤로가기를 누르면 미니앱이 정상 종료되어야 한다.
- 하단 탭을 사용하면 토스 앱과 어울리는 플로팅 형태로 제공한다.
- 미니앱 진입 직후 바텀시트나 모달을 자동으로 열지 않는다.
- CTA 문구는 다음 동작을 명확히 설명한다.
- 사용자 확인과 동의가 필요한 화면은 가능한 범위에서 TDS 컴포넌트를 사용한다.
- 라이트 모드를 기본으로 하고 모바일 최소 폭에서 잘림이나 가로 스크롤이 없어야 한다.
- 주요 동작은 2초 안에 반응하며 로딩, 빈 상태, 오류 상태를 제공한다.
- 키보드 포커스, 레이블, 명도 대비 등 접근성을 검수한다.
- 지도 등 필수 사유가 없으면 제스처 확대·축소를 비활성화한다.

## 7. 기술 및 보안

- WebView 앱은 CSR 또는 SSG로 제공하며 SSR을 사용하지 않는다.
- `eval`, 동적 외부 코드 실행 및 신뢰할 수 없는 코드 실행 기능을 사용하지 않는다.
- 브라우저 히스토리를 조작해 외부 자사 사이트로 보내지 않는다.
- 실시간 통신은 `wss://`만 사용한다.
- 비밀키, mTLS 인증서, 서버 API 자격증명을 클라이언트 번들에 포함하지 않는다.
- 권한 요청 전에 목적을 안내하고, 거절해도 핵심 기능은 계속 사용할 수 있어야 한다.
- 비정상적인 네트워크·메모리 사용과 중복 요청을 방지한다.

## 8. 알림

- 알림 동의 요청은 실제 발송 기능과 구분해서 설명한다.
- 알림을 거절한 사용자도 오늘의 할 일과 기록 기능을 사용할 수 있어야 한다.
- 로그인과 서버가 없는 단계에서는 앱 진입 시 인앱 리마인더만 제공한다.
- 토스 푸시는 사용자 식별키, 알림 동의, 메시지 검수, mTLS 파트너 서버 및 발송 스케줄러가 준비된 후 적용한다.
- 알림에는 민감한 집안 환경이나 구성원 활동 내용을 과도하게 노출하지 않는다.

## 9. 생성형 AI

현재 규칙 기반 추천은 생성형 AI 결과로 표시하지 않는다. 추후 생성형 AI가 추천, 요약, 안내 또는 자동 응답을 만들면 아래 기준을 적용한다.

- 최초 AI 기능 이용 전에 생성형 AI 사용 사실을 알린다.
- AI가 만든 결과마다 즉시 인지할 수 있는 `AI 생성` 라벨이나 배지를 표시한다.
- AI 결과를 사실이나 전문가 판단처럼 단정하지 않는다.
- 위험한 세제 혼합, 전기·가스·화재·위생 관련 안내를 차단하거나 검증된 고정 안내로 대체한다.
- 모델과 핵심 프롬프트 변경 시 안전성 회귀 테스트와 앱 배포 검토 필요 여부를 확인한다.
- 부적절한 결과를 신고하고 즉시 수정·차단할 운영 절차를 마련한다.

## 10. 공동 집과 통계

- 초대 링크나 코드는 집 참여 목적으로만 사용하고 외부 서비스 가입을 유도하지 않는다.
- 집 구성원만 해당 집의 일정과 기록을 조회할 수 있어야 한다.
- 완료 기록은 실제 수행자와 수행 시각을 감사 가능하게 저장한다.
- 구성원 통계는 기여도를 돕는 정보로 제공하며 성실성, 관계의 가치 또는 인격을 평가하는 지표로 표현하지 않는다.
- 공개 순위, 꼴찌 강조, 과도한 경쟁 유도를 기본 기능으로 제공하지 않는다.
- 구성원 제거, 탈퇴, 소유권 이전과 데이터 접근 차단을 검증한다.

## 11. 기능 완료 기준

새 기능은 아래 조건을 모두 만족해야 완료로 처리한다.

1. 제품 명세와 데이터 정책이 문서화되어 있다.
2. 외부 이동, 로그인, 결제, 광고, AI, 권한 사용 여부를 확인했다.
3. 정상·빈 상태·오류·거절 흐름을 구현했다.
4. 접근 가능한 이름과 키보드 포커스를 제공한다.
5. TypeScript 검사와 웹 프로덕션 빌드가 성공한다.
6. 앱인토스 `.ait` 번들 빌드가 성공한다.
7. 토스 샌드박스에서 내비게이션, 뒤로가기, 앱 내 기능을 검증한다.
8. 정책 체크리스트에 미완료 항목과 출시 차단 여부를 기록한다.

## 13. 정책 갱신 규칙

- 출시 준비 단계와 주요 SDK 업데이트 때 공식 정책을 다시 확인한다.
- 공식 문서와 이 문서가 충돌하면 공식 문서를 우선한다.
- 정책 변경은 코드 변경과 동일하게 검토하고 Git 이력에 남긴다.
- 정책 위반 가능성이 있는 기능은 구현을 중단하고 제품 책임자의 결정을 받는다.
