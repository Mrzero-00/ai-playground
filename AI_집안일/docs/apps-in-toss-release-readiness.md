# 앱인토스 출시 준비 체크리스트

기준일: 2026-07-15

이 문서는 앱인토스 공식 서비스 오픈 정책, 비게임 출시 가이드, 미니앱 출시 가이드를 기준으로 `AI 집안일`의 출시 준비 상태와 담당자를 구분한다.

## 현재 결론

- 웹 프로덕션 빌드와 `.ait` 번들 생성은 성공한다.
- 현재 생성 번들: `ai-housework.ait` (약 3.7MB, 압축 해제 100MB 제한 이내로 예상되나 콘솔 업로드 시 재확인)
- 현재 `appName`은 임시값 `ai-housework`, 브랜드 아이콘 URL은 비어 있다.
- 브라우저 익명 쿠키로 사용자를 식별하고 있어 앱인토스 비게임 사용자 식별키가 아직 연동되지 않았다.
- 약관, 개인정보처리방침, 고객 문의 경로, 탈퇴/데이터 삭제 기능이 없다.
- 앱인토스 비게임 내비게이션, 딥링크, 뒤로가기, 실제 토스 앱/샌드박스 검증이 남아 있다.
- 따라서 현재 상태는 **번들 생성 가능 / 출시 검수 요청 불가**다.

## 먼저 결정할 출시 범위

첫 출시에서는 토스 로그인 대신 비게임용 `getAnonymousKey`를 사용한다. 별도 동의·계약 없이 미니앱별 고유 사용자를 식별할 수 있고, 현재 집안일 데이터 저장 구조에도 연결하기 쉽다. 토스 로그인, 기능성 푸시, 토스페이까지 첫 출시 범위에 넣으면 사업자 등록, 계약, mTLS 서버 운영이 추가되므로 후속 버전으로 분리한다.

토스쇼핑 제휴 구매는 첫 검수 전에 제휴 방식과 외부 이동 허용 여부를 토스에 확인한다. 승인 전에는 비활성 상태로 유지하거나 출시 번들에서 숨긴다.

## 권장 진행 순서

1. 사용자가 콘솔에서 앱을 만들고 `appName`, 앱 이름, 아이콘, 문의 이메일을 전달한다.
2. 개발 쪽에서 사용자 식별키, 내비게이션, 약관/삭제 화면, 딥링크를 구현한다.
3. 사용자가 약관·개인정보처리방침 URL과 토스쇼핑 정책 문의 결과를 전달한다.
4. 개발 쪽에서 운영 Origin, 외부 링크 정책, 서버 인증을 확정하고 출시 후보 번들을 만든다.
5. 사용자가 샌드박스에서 실기기 QA를 수행하고 결과를 공유한다.
6. 개발 쪽에서 발견된 문제를 수정한 뒤 최종 `.ait`를 전달한다.
7. 사용자가 콘솔에서 검수 요청 및 승인 후 출시를 진행한다.
