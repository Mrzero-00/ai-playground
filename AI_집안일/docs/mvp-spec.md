# AI 집안일 MVP 제품 요구사항

## 1. 문서 목적

로그인 없이 한 기기에서 사용할 수 있는 **Phase 1 로컬 MVP** 범위를 정의한다. 사용자가 집 프로필을 설정하면 맞춤 집안일을 추천받고, 추천 또는 직접 만든 집안일을 반복 일정으로 관리하며, 오늘 할 일을 완료하고 기록을 확인할 수 있어야 한다.

이 문서의 `HomeProfile`은 Phase 1에서 현재 기기에 저장되는 단일 집 프로필이다. 개인 계정 `User`와 공동 생활 공간 `Home`을 분리하고 구성원을 초대하는 기능은 로그인·서버 저장소가 도입되는 Phase 2 범위이며, 상세 내용은 [공동 집 Phase 2 명세](./shared-home-spec.md)를 따른다.

## 2. MVP 목표와 제외 범위

### 목표

- 최초 사용자가 3분 안에 집 프로필과 기본 집안일 목록을 만들 수 있다.
- 사용자가 오늘 해야 할 일을 한 화면에서 파악하고 완료할 수 있다.
- 사용자가 원하는 집안일과 반복 주기를 직접 추가할 수 있다.
- 재방문 시 프로필, 일정, 완료 기록이 기기에 유지된다.
- 알림을 허용한 사용자에게 할 일이 있는 시점을 안내한다.

### 이번 버전에서 제외

- 로그인, 서버 동기화, 기기 간 데이터 이전
- 가족·동거인 초대와 담당자 배정
- 개인 계정과 여러 Home의 분리, 공동 완료 및 구성원별 통계
- 집안일 레벨, 주부력 점수, 랭킹 및 공유
- AI 모델을 호출하는 동적 추천
- 사진 인증, 메모, 미루기, 건너뛰기
- 결제 및 운영자용 관리 화면

점수화와 AI 추천은 충분한 실제 수행 기록을 확보한 다음 단계에서 설계한다. 이번 추천은 설명 가능한 규칙 기반으로 제공한다.

### Phase 경계

| 구분 | Phase 1 로컬 MVP(현재) | Phase 2 공동 집 |
| --- | --- | --- |
| 식별 | 계정 없음, 기기 1대 | 로그인한 개인 User |
| 집 | 로컬 HomeProfile 1개 | 서버 Home 1개 이상, User와 분리 |
| 구성원 | 지원하지 않음 | 초대 코드·링크로 참여 |
| 담당 | 모두 개인 할 일 | 개인 담당·복수 담당·공동 처리 |
| 완료 기록 | 완료 시각과 집안일 스냅샷 | 완료한 User와 Home까지 감사 기록 |
| 통계 | 개인 기기의 전체 완료 내역 | 집 전체 및 구성원별 기여 통계 |
| 저장 | 로컬 저장소 | 서버를 원본으로 사용하고 기기 캐시 허용 |

Phase 1 구현에서 가상의 구성원이나 임시 계정을 만들지 않는다. Phase 2 전환 시 사용자가 로그인한 뒤 로컬 프로필·집안일·완료 기록을 새 Home으로 가져올지 명시적으로 선택하게 하며, 자동 업로드하지 않는다.

## 3. 핵심 사용자 흐름

### 3.1 최초 진입

1. 서비스 소개 화면에서 `시작하기`를 누른다.
2. 집 형태, 가구원 수, 반려동물 정보를 입력한다.
3. 추천 결과에서 사용할 집안일을 선택한다.
4. 선택한 항목이 내 집안일에 저장되고 오늘 화면으로 이동한다.
5. 알림의 효용을 설명한 뒤 권한을 요청한다. 권한 거절 시에도 서비스를 계속 이용할 수 있다.

### 3.2 재방문

1. 저장된 프로필이 있으면 오늘 화면으로 바로 진입한다.
2. 오늘 예정된 미완료 항목과 완료 항목을 구분해 본다.
3. 집안일을 완료하면 즉시 완료 상태와 오늘 진행률이 갱신된다.
4. 기록 화면에서 날짜별 완료 내역을 확인한다.

### 3.3 집안일 추가

1. 집안일 관리 화면에서 `직접 추가`를 누른다.
2. 이름과 반복 유형을 입력하고 세부 반복 값을 설정한다.
3. 첫 예정일과 알림 여부를 정하고 저장한다.
4. 첫 예정일이 오늘이면 오늘 할 일에 즉시 나타난다.

### 3.4 프로필 변경

1. 프로필 화면에서 집 정보를 수정한다.
2. 새 조건에만 해당하는 추천 항목을 보여준다.
3. 사용자가 추가할 항목만 선택한다.
4. 기존 내 집안일은 자동 삭제하거나 주기를 덮어쓰지 않는다.

## 4. 화면 및 기능 요구사항

### 4.1 집 프로필

필수 입력:

- 집 형태: 원룸·오피스텔, 아파트, 빌라·다세대, 단독주택, 기타 중 1개
- 가구원 수: 1명, 2명, 3명, 4명, 5명 이상 중 1개
- 가구 특성: 혼자 거주, 부부·커플, 자녀와 거주, 룸메이트와 거주 중 1개
- 반려동물: 없음, 반려견, 반려묘, 기타 중 복수 선택

동작:

- 필수 입력을 완료해야 다음 단계로 이동할 수 있다.
- `기타`는 MVP에서 자유 입력을 요구하지 않고 일반 추천만 적용한다.
- 프로필은 언제든 수정할 수 있다.
- 로그인 전까지 모든 정보는 현재 기기에만 저장된다는 안내를 표시한다.

### 4.2 추천 집안일 선택

- 공통 추천과 프로필 맞춤 추천을 합쳐 중복 없이 표시한다.
- 각 항목에 이름, 권장 주기, 추천 이유를 표시한다.
- 초기에는 모든 추천 항목을 선택 상태로 제공하되 개별 해제가 가능하다.
- 저장 시 선택된 템플릿을 `내 집안일`로 복사한다.
- 이미 동일 템플릿이 추가되어 있으면 중복 생성하지 않는다.
- 사용자가 저장 후 이름, 반복 주기, 알림을 자유롭게 수정할 수 있다.

### 4.3 집안일 관리

- 활성 집안일을 목록으로 표시한다.
- 항목의 이름, 반복 요약, 다음 예정일, 알림 상태를 표시한다.
- 추가, 수정, 비활성화, 삭제를 지원한다.
- 삭제 전 확인을 받고 기존 완료 기록은 유지한다.
- 비활성화하면 새 할 일을 만들지 않되 기존 기록은 유지한다.
- 템플릿 기반 항목도 수정 후에는 사용자 설정을 우선한다.

커스텀 집안일 입력:

- 이름: 필수, 공백 제외 1~30자
- 반복 유형: 매일, 매주, 매월, 매년, 사용자 간격 중 1개
- 매주: 월~일 중 1개 이상 복수 선택
- 매월: 1~31일 중 1개 선택
- 매년: 월·일 1개 선택
- 사용자 간격: 일·주·개월 단위와 1~365 사이 숫자
- 첫 예정일: 필수, 오늘 이후
- 알림 사용 여부 및 알림 시각

반복 계산 원칙:

- 매월 29~31일이 없는 달에는 해당 월의 마지막 날을 예정일로 사용한다.
- 매년 2월 29일은 윤년이 아닌 해에 2월 마지막 날로 처리한다.
- 완료 시점을 기준으로 밀지 않고 원래 달력 반복 규칙을 유지한다.
- 같은 집안일에 과거 미완료 일정이 여러 개 생겨도 오늘 화면에는 가장 오래된 1건만 표시한다. 완료하면 다음 미완료 예정 건을 노출한다.
- 시간대는 기기 로컬 시간대를 사용하고 날짜 경계는 자정이다.

### 4.4 오늘 할 일

- 오늘 예정 및 오늘 이전부터 미완료인 활성 항목을 표시한다.
- `미완료`, `완료` 두 구역으로 구분한다.
- 오래 밀린 항목에는 예정일을 표시한다.
- 완료 버튼을 누르면 완료 시각을 기록하고 완료 구역으로 이동한다.
- 당일 완료를 취소할 수 있으며 취소하면 완료 기록을 제거하고 미완료로 되돌린다.
- 오늘 진행률은 `오늘 표시된 항목 중 완료 수 / 오늘 표시된 전체 항목 수`로 계산한다.
- 할 일이 없으면 집안일 추가 및 관리 화면으로 가는 빈 상태를 제공한다.

### 4.5 완료 기록

- 날짜별 완료 건을 최신순으로 표시한다.
- 각 기록에 집안일 이름, 예정일, 실제 완료 시각을 표시한다.
- 집안일이 수정 또는 삭제되어도 완료 당시 이름을 기록에 보존한다.
- MVP 조회 범위는 전체이며 월별 구분 또는 월 선택을 제공한다.
- 기록 화면에서는 완료 취소를 제공하지 않는다. 오늘 완료 건의 취소는 오늘 화면에서만 가능하다.

### 4.6 알림

- 알림 권한은 온보딩 완료 후 맥락을 설명하고 요청한다.
- 집안일별로 알림 사용 여부와 시각을 설정한다.
- 기본 알림 시각은 오전 9시이며 사용자가 변경할 수 있다.
- 같은 날짜·시각의 여러 항목은 가능하면 `오늘 할 집안일 N개가 있어요` 형태로 묶는다.
- 알림을 누르면 오늘 화면으로 이동한다.
- 완료된 항목의 해당 일정 알림은 취소한다.
- 권한이 없거나 실행 환경에서 예약 알림을 지원하지 않으면 오늘 화면 내 안내만 제공하고 설정에 상태를 명시한다.
- 권한 요청 거절을 오류로 취급하지 않으며 반복적으로 권한 창을 띄우지 않는다.

## 5. 추천 규칙

추천 엔진은 `공통 템플릿 + 조건별 템플릿`의 합집합이다. 동일 `templateId`는 한 번만 표시한다. 여러 조건에 해당해도 가장 짧은 권장 주기를 자동 적용하지 않고 템플릿에 정의된 기본값을 사용한다. 추천 이유만 조건에 맞게 합친다.

## 6. 데이터 모델

로그인 전 로컬 저장소를 사용한다. 모든 주요 엔터티는 `id`, `createdAt`, `updatedAt`을 가진다. 날짜는 `YYYY-MM-DD`, 시각은 로컬 `HH:mm`, 이벤트 시각은 ISO 8601 문자열로 저장한다.

### HomeProfile

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| id | string | 로컬 프로필 ID. MVP에서는 1개 |
| housingType | enum | studio, apartment, multiFamily, detached, other |
| householdSize | number | 1~5, 5는 5명 이상 |
| householdType | enum | single, couple, children, roommate |
| pets | enum[] | dog, cat, other. 빈 배열은 없음 |
| onboardingCompleted | boolean | 최초 설정 완료 여부 |
| notificationPrompted | boolean | 알림 권한 요청 경험 여부 |

### ChoreTemplate

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| id | string | 고정 템플릿 ID |
| name | string | 기본 표시 이름 |
| reason | string | 추천 이유 |
| conditions | object[] | 적용할 프로필 조건. 빈 배열은 공통 |
| defaultRecurrence | RecurrenceRule | 권장 반복 규칙 |
| defaultNotificationEnabled | boolean | 기본값 |

### Chore

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| id | string | 내 집안일 ID |
| sourceTemplateId | string \| null | 커스텀은 null |
| name | string | 현재 이름 |
| recurrence | RecurrenceRule | 현재 반복 규칙 |
| startDate | date | 첫 예정일 |
| nextDueDate | date | 계산된 다음 미완료 예정일 |
| notificationEnabled | boolean | 항목별 알림 |
| notificationTime | time \| null | 알림 시각 |
| active | boolean | 새 일정 생성 여부 |

### RecurrenceRule

| 필드 | 타입 | 설명 |
| --- | --- | --- |

### Completion

| 필드 | 타입 | 설명 |

| defaultNotificationTime | time | 기본 09:00 |

별도 `Task` 엔터티는 MVP에서 영구 저장하지 않는다. 오늘 할 일은 활성 `Chore`, 반복 규칙, `Completion`을 기준으로 계산한다. 이는 장기간 미접속 시 불필요한 일정 인스턴스가 대량 생성되는 것을 피한다.

## 7. 상태 및 계산 규칙

- 예정 건 식별 키는 `choreId + dueDate`다.
- 해당 키의 Completion이 있으면 완료, 없으면 미완료다.
- 오늘 목록은 `nextDueDate <= today`인 활성 Chore로 구성한다.
- 완료 후 반복 규칙에서 다음 예정일을 계산해 `nextDueDate`를 갱신한다.
- 완료 취소 후 취소한 예정일과 현재 다음 예정일 중 빠른 날짜를 `nextDueDate`로 복원한다.
- 반복 규칙이나 시작일을 수정하면 수정일 이후 일정만 새 규칙을 따르고 기존 완료 기록은 유지한다.
- 로컬 데이터 파싱에 실패하면 앱 전체가 중단되지 않도록 빈 초기 상태와 복구 안내를 제공한다.

## 8. 완료 조건

다음 시나리오가 모두 동작하면 MVP 제품 요구사항을 충족한다.

1. 새 사용자가 프로필을 저장하고 조건에 맞는 추천을 선택해 오늘 화면에 진입한다.
2. 앱을 종료하고 다시 열어도 프로필, 내 집안일과 완료 기록이 유지된다.
3. 매일·매주·매월·매년·사용자 간격 집안일을 각각 생성하고 다음 예정일이 올바르게 계산된다.
4. 오늘 및 밀린 집안일을 완료하고 취소할 수 있으며 진행률과 기록이 즉시 반영된다.
5. 프로필 수정 후 새 추천을 추가해도 기존 사용자 설정과 기록이 보존된다.
6. 알림 허용, 거절, 미지원 환경 모두에서 앱의 핵심 기능이 정상 작동한다.
7. 31일 월간 반복과 2월 29일 연간 반복의 예외가 정의된 규칙대로 처리된다.
8. 추천 항목과 커스텀 항목을 수정·비활성화·삭제해도 과거 완료 기록이 남는다.
