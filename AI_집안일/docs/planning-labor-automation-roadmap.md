# 기획노동 자동화 제품 방향

## 최우선 비전

AI 집안일의 최종 목표는 사용자가 집안일을 기억하고 계획하는 데 에너지를 쓰지 않도록 하는 것이다. 앱이 집의 상태와 생활 패턴을 바탕으로 **기억, 판단, 일정 생성, 담당자 제안, 알림, 누락 확인, 재조정**을 맡고, 사용자는 지금 해야 할 가사노동을 명확하게 안내받아 실행에 집중할 수 있어야 한다.

기획노동 점수는 사용자를 평가하기 위한 기능이 아니다. 자동화 이전에는 부담이 어디에 몰려 있는지 발견하고 대화하는 도구이며, 자동화 이후에는 앱이 실제로 기획 부담을 얼마나 줄였는지 측정하는 제품 지표로 사용한다.

## 단계별 발전

### 1. 측정과 역할 분리

- 구성원별 기획 부담과 가사 실행 부담 자가 진단
- 집안일마다 계획 담당자와 실행 담당자 분리
- 진단 성향과 현재 배정량을 반영한 자동 분배 및 전체 목록 모드 제공
- 집 단위 기획노동 지수와 진단 참여율 표시
- 비교나 서열 대신 부담 불균형을 발견하는 표현 사용

### 2. 규칙 기반 기획 자동화

- 집 형태, 구성원, 반려동물, 계절, 반복 주기를 기반으로 일정 자동 생성
- 기한과 최근 수행 이력을 바탕으로 우선순위 자동 결정
- 구성원의 선호와 최근 부담을 고려한 담당자 제안
- 미완료 시 비난 없이 일정 재조정

### 3. 상황 인지형 자동 운영

- 소모품 교체, 계절성 관리, 장기 미수행 작업을 앱이 선제적으로 발견
- 구성원 일정과 수행 패턴을 반영해 알림 시점 최적화
- 앱이 제안한 계획을 사용자가 승인·수정할 수 있는 통제권 제공
- 자동화 근거와 변경 내역을 이해하기 쉽게 설명

### 4. 기획노동 최소화

- 사용자가 매번 할 일을 등록하거나 담당자를 정하지 않아도 집이 운영되는 상태
- 앱이 예외 상황을 감지하고 대안을 제안
- 사용자는 오늘의 실행 목록만 확인하고 완료 처리
- 기획노동 절감률, 누락 감소, 부담 균형 개선을 핵심 성과로 측정

## 제품 원칙

- 성별, 관계, 소득, 가족 형태로 역할을 추정하지 않는다.
- 자동 배분은 강제가 아닌 제안이며 구성원이 언제든 변경할 수 있다.
- 자가 진단은 능력, 성실성, 애정의 점수가 아니다.
- 민감한 생활 패턴은 필요한 범위만 저장하고 삭제·초기화 경로를 제공한다.
- 생성형 AI를 도입할 때는 앱인토스 정책에 맞는 고지, 결과 라벨, 안전 필터, 운영 대응을 먼저 설계한다.
