# 공동 집 Phase 2 제품 명세

## 1. 목적과 범위

로그인한 개인 `User`와 함께 관리하는 생활 공간 `Home`을 분리하고, 가족·동거인이 한 Home의 집안일을 배정·공동 처리하며 완료 주체와 기여 현황을 확인하는 Phase 2를 정의한다.

포함 범위:

- 개인 계정과 Home의 분리 및 한 User의 복수 Home 참여
- 초대 코드와 링크 생성·참여·취소·만료
- 소유자와 구성원 권한
- 집안일의 개인 담당, 복수 담당, 공동 처리
- 완료한 구성원 감사 기록과 구성원별 통계
- Phase 1 로컬 데이터의 선택적 가져오기 진입점

제외 범위:

- 공개 랭킹, 소셜 피드, 타 Home 간 비교
- 미성년자 계정에 특화된 보호자 동의
- 세분화된 관리자 직책, 게스트·읽기 전용 역할
- 집안일별 점수·난이도 보정과 집안일 레벨
- 구성원 간 금전 정산 또는 보상

## 2. 핵심 원칙

- 계정은 사람을, Home은 공동 생활 공간과 그 데이터를 나타낸다.
- User의 개인 프로필은 Home 구성원에게 필요한 최소 정보만 노출한다.
- 담당은 책임을 명확히 하되, 다른 구성원의 도움이나 공동 완료를 막지 않는다.
- 통계는 감시·서열화보다 분담 상황을 이해하고 대화하는 도구로 제공한다.
- 완료 기록의 주체·시각 변경은 추적 가능해야 한다.

## 3. 사용자 흐름

### 3.1 새 Home 만들기

1. 로그인한 사용자가 Home 이름과 집 프로필을 입력한다.
2. 생성자는 소유자(Owner) Membership을 얻는다.
3. Phase 1 로컬 데이터가 있으면 `가져오기` 또는 `새로 시작`을 선택한다.
4. 가져오기를 선택하면 대상 항목과 완료 기록 수를 미리 보여주고 확인 후 업로드한다.
5. 추천 집안일을 선택하거나 기존 집안일 관리 화면으로 이동한다.

### 3.2 구성원 초대

1. 소유자가 구성원 관리에서 초대를 생성한다.
2. 서비스가 만료 시각이 있는 1회용 코드와 딥링크를 함께 제공한다.
3. 초대받은 사용자는 로그인 후 Home 이름, 소유자 표시명, 현재 구성원 수를 확인한다.
4. `참여하기`를 명시적으로 누르면 Membership이 생성되고 초대는 사용 완료된다.
5. 이미 참여 중인 사용자는 중복 참여하지 않고 해당 Home으로 이동한다.

### 3.3 담당 설정과 수행

1. 권한이 있는 사용자가 집안일을 만들거나 수정하면서 처리 방식을 선택한다.
2. `담당 없음`, `개인/복수 담당`, `모두 함께` 중 하나를 지정한다.
3. 오늘 화면은 내 담당을 우선 표시하고 Home의 전체 할 일도 별도로 볼 수 있게 한다.
4. 구성원 누구나 허용된 항목을 완료하며 실제 완료자는 로그인 User로 기록된다.
5. `모두 함께` 항목은 참여자를 복수 선택해 한 번에 공동 완료하거나 각자 참여를 추가할 수 있다.

### 3.4 구성원 변경과 탈퇴

- 소유자는 구성원을 내보낼 수 있다. 내보낸 즉시 새 데이터 접근 권한을 잃는다.
- 구성원은 스스로 탈퇴할 수 있다. 과거 완료 기록의 표시명 스냅샷과 익명화되지 않은 감사 ID는 서버 정책에 따라 보존한다.
- 담당자가 탈퇴하거나 제거되면 해당 배정은 해제하고 소유자에게 알린다.
- 마지막 소유자는 다른 구성원에게 소유권을 이전하기 전 탈퇴할 수 없다.
- Home 삭제는 소유자만 가능하며 재확인과 유예/복구 정책을 거친다.

## 5. 담당과 공동 처리 규칙

`assignmentMode`:

- `unassigned`: 특정 담당자 없음. 모든 구성원이 볼 수 있고 누구나 완료 가능하다.
- `assigned`: 한 명 이상의 활성 구성원이 담당한다. 담당자에게 우선 노출·알림하지만 다른 구성원도 `대신 완료`로 처리할 수 있다.
- `shared`: Home 전체가 함께하는 일이다. 완료 시 한 명 이상의 참여자를 기록한다.

공통 규칙:

- 담당자 지정은 Membership ID를 사용하며 Home 밖 User를 지정할 수 없다.
- 담당자가 0명이 된 `assigned` 항목은 자동으로 `unassigned`로 전환한다.
- 하나의 예정 건은 한 번만 완료 상태가 된다. 여러 사용자의 동시 완료 요청은 서버의 고유 키로 중복을 막고, 늦은 요청에는 이미 완료한 사람을 안내한다.
- `assigned`에서 실제 완료자가 담당자가 아닐 수 있으며 통계에는 실제 완료자만 반영한다.
- `shared` 완료는 1개의 Completion과 여러 CompletionParticipant로 표현한다. 참여자는 완료 후 제한된 시간(기본 24시간) 동안 본인을 추가할 수 있다.
- 알림은 담당자 또는 shared의 알림 수신 구성원에게만 보내고, 완료되면 해당 예정 건의 남은 알림을 취소한다.

## 7. 초대 수명주기와 보안

1. `생성`: 소유자가 생성하며 기본 만료는 24시간, 1회 사용이다.
2. `공유`: 원문 토큰은 링크에만 포함하고 서버에는 해시로 저장한다. 짧은 코드는 충분한 엔트로피와 시도 제한을 적용한다.
3. `검증`: 로그인 후 상태, 만료, Home 상태, 기존 Membership을 원자적으로 확인한다.
4. `수락`: Membership 생성과 Invitation의 accepted 전환을 하나의 트랜잭션으로 처리한다.
5. `취소`: 소유자가 수락 전 언제든 revoked로 바꿀 수 있다.
6. `만료`: expiresAt 이후 서버가 expired로 취급하며 재사용할 수 없다.

추가 규칙:

- 코드 입력은 계정·IP·기기 단위 속도 제한을 둔다.
- 초대 미리보기에는 주소, 집안일, 수행 기록을 노출하지 않는다.
- 링크가 유출되었다면 소유자는 즉시 취소하고 새 초대를 발급할 수 있다.
- 수락 후 뒤로 가거나 재호출해도 중복 Membership이 생기지 않아야 한다.
- 초대 전달 대상의 이메일·전화번호는 이번 범위에서 수집하지 않는다.

## 8. 통계 정의

기본 기간은 이번 주이고 최근 4주·이번 달·직접 선택을 지원한다. Home 시간대의 예정일과 완료 시각을 기준으로 계산하며 `voided` 완료는 제외한다.

### 집 전체

- 예정 건수: 기간 내 dueDate가 있는 활성 또는 당시 활성 집안일의 고유 예정 건 수
- 완료 건수: 기간 내 예정 건 중 활성 Completion이 있는 수
- 완료율: `완료 건수 / 예정 건수 × 100`. 예정 0건이면 숫자 대신 `—` 표시
- 정시 완료율: 완료 건 중 `completedAt의 Home 현지 날짜 <= dueDate`인 비율
- 밀린 건수: 오늘 이전 dueDate이며 활성 Completion이 없는 건 수

### 구성원별

- 개인 완료 건수: 그 Membership이 실제 참여한 완료 수. 일반 완료는 completedBy, shared는 participant 기준
- 기여 비율: `개인 완료 가중치 합 / 전체 구성원의 완료 가중치 합 × 100`
- 담당 이행률: 해당 구성원이 담당자였던 예정 건 중 기간 내 완료된 비율. 누가 대신 완료했더라도 집안일 자체가 완료되면 이행으로 보되 `대신 완료`를 별도 표기
- 대신 완료 건수: 담당자가 아닌 구성원이 완료한 assigned 건 수

공동 완료 1건의 기여 가중치는 참여자 수로 균등 분할한다. 예: 2명이 함께 완료하면 각 0.5건이다. `unassigned`와 일반 `assigned` 완료는 실제 완료자에게 1건을 준다. 담당 이행률과 개인 기여는 서로 다른 지표이며 합쳐 단일 점수로 만들지 않는다.

통계에 포함되는 멤버십은 기간 중 활동한 구성원이다. 탈퇴·제거된 구성원의 과거 기여는 `이전 구성원` 또는 보존된 표시명으로 남기되 현재 구성원으로 오인되지 않게 표시한다.

## 9. 개인정보와 공정성 원칙

- Home 데이터는 활성 구성원만 접근하며 모든 조회·쓰기에서 homeId 소속을 서버가 검증한다.
- 초대 전 미리보기와 푸시 알림에 민감한 집 프로필·상세 수행 기록을 노출하지 않는다.
- 구성원에게 개인 이메일, 전화번호, 로그인 방식 같은 계정 정보를 공개하지 않는다.
- 탈퇴 시 접근은 즉시 차단하고 계정 삭제·Home 삭제·완료 감사 기록 보존 기간을 개인정보 처리방침에 명시한다.
- 데이터 내보내기와 삭제 요청 경로를 제공하고, 공동 데이터 삭제가 다른 구성원 기록에 미치는 영향을 사전에 설명한다.
- 구성원 통계는 Home 내부에서만 기본 공개하며 외부 공유는 별도 동의 기능 전에는 제공하지 않는다.
- 완료량만으로 성실성·가정성·관계 기여도를 단정하지 않는다. 집안일 난이도, 돌봄, 보이지 않는 노동, 개인의 건강·시간 차이를 통계가 완전히 반영하지 못함을 안내한다.
- 구성원 순위, 꼴찌 강조, 비난성 알림, 강제 목표를 제공하지 않는다.
- 기여 비율은 업무 분담 대화를 위한 관찰값이며 보상·불이익의 자동 근거로 사용하지 않는다.
- 표시명 변경·탈퇴 이후에도 감사 기록의 연결성을 유지하되 UI 노출은 최소화한다.

## 10. 오류와 경계 상황

- 초대 수락과 동시에 소유자가 취소하면 서버 트랜잭션에서 먼저 확정된 한 상태만 유효하다.
- 오프라인 완료는 임시 요청 ID로 저장하고 재연결 시 멱등 처리한다. 다른 사용자가 먼저 완료했다면 중복 완료를 만들지 않는다.
- 구성원이 보고 있는 동안 제거되면 다음 서버 요청에서 권한 오류를 받고 Home 선택 화면으로 이동한다.
- Home 시간대 변경은 이후 일정 계산에 적용하되 기존 dueDate와 완료 기록을 소급 변경하지 않는다.
- 반복 집안일 담당 변경은 다음 미완료 예정 건부터 적용하고 이미 완료된 기록의 담당 스냅샷은 유지한다.

## 11. 인수 조건

1. User A가 Home을 만들면 owner Membership이 생성되고 User와 Home의 ID가 분리된다.
2. A가 만든 코드와 링크로 로그인한 User B가 한 번만 참여하며, 재시도해도 중복 Membership이 생기지 않는다.
3. 만료·취소·이미 사용한 초대는 참여에 실패하고 적절한 복구 안내를 표시한다.
4. 구성원은 초대 생성·제거·Home 삭제를 수행할 수 없고 서버에서도 거부된다.
5. 한 명·복수 담당과 shared 집안일을 만들고 오늘 화면에서 내 담당 및 전체 항목을 구분해 볼 수 있다.
6. 비담당자의 대신 완료, 공동 완료의 복수 참여자, 동시 완료 요청이 정의된 규칙대로 단일 Completion에 기록된다.
7. 완료 기록에서 누가 언제 완료·참여·취소했는지 확인할 수 있고 집안일이나 구성원 변경 후에도 당시 스냅샷이 유지된다.
8. 소유자가 구성원을 제거하면 즉시 접근이 차단되고 담당이 안전하게 해제되며 과거 통계는 보존된다.
9. 마지막 소유자는 소유권 이전 없이 탈퇴할 수 없다.
10. 집 전체 완료율, 정시 완료율, 밀린 건수와 구성원별 완료·기여·담당 이행·대신 완료가 예시 데이터로 정확히 계산된다.
11. Phase 1 데이터는 로그인만으로 자동 업로드되지 않고, 대상 Home과 가져올 범위를 확인한 뒤에만 저장된다.
12. 다른 Home ID로 직접 API를 호출해도 데이터가 노출·수정되지 않는다.
