# Alpine Mercenaries 프로젝트 가이드

## 목표

- 최대 5인이 플레이하는 세션형 3D 3인칭 협동 액션 RPG를 만든다.
- 제작 기본 시점은 캐릭터 뒤의 숄더뷰이며 쿼터뷰는 비교 검증 모드로만 유지한다.
- 무기가 직업을 대신하며, 각 역할은 수치가 아니라 위치·방향·타이밍으로 수행한다.
- 하나의 용병 사무실을 소셜 허브로 사용하고 의뢰별 전투 맵을 인스턴스로 제공한다.
- 장기적으로 Unreal Engine과 Steam 출시를 목표로 한다.

## 현재 기술 단계

- `prototype/`가 아닌 루트의 Vite + Three.js 코드는 메커니즘 검증용이다.
- 프로토타입에 최종 아트 자산이나 제작판 네트워크 코드를 결합하지 않는다.
- 제작판으로 검증된 규칙을 이식하기 전, `docs/`의 수용 기준을 갱신한다.

## 명령

- 개발 서버: `pnpm dev`
- 정적 빌드: `pnpm build`
- 빌드 미리보기: `pnpm preview`
- 정적 검사: `pnpm check`

## 단계 완료와 GitHub

- 각 개발 단계가 끝나면 관련 테스트, 정적 검사와 빌드를 먼저 통과시킨다.
- 해당 단계와 직접 관련된 파일만 별도 커밋한다. 워크스페이스의 다른 프로젝트 변경은 포함하지 않는다.
- 커밋 메시지에는 완료한 단계와 결과를 드러낸다.
- 단계별 커밋을 `origin`의 현재 작업 브랜치에 푸시한 뒤에만 해당 단계를 완료로 보고한다.
- 푸시가 인증, 네트워크 또는 원격 충돌로 실패하면 성공했다고 표현하지 않고 원인을 기록한다.

## 설계 원칙

- 기존 게임의 고유 캐릭터, 몬스터, 의상, 명칭을 복제하지 않는다.
- 방패 방어는 뒤의 아군까지 실제 공간 판정으로 보호해야 한다.
- 유니크 무기는 단순 수치 상승보다 새로운 행동을 제공한다.
- 모든 아이템과 재화는 거래 가능하며 귀속 상태를 만들지 않는다.
- 불멸화된 장비는 현재 상태와 최대 내구도가 모두 감소하지 않는다.
- 온라인 판정, 아이템 소유권, 거래는 최종적으로 서버 권위 모델을 사용한다.
- 전투 맵은 배경만 바꾸지 않고 이동, 환경 위험, 몬스터, 목표를 함께 바꾼다.

## 코드 원칙

- 게임 규칙과 Three.js 표현을 가능한 한 분리한다.
- 입력, 플레이어, 몬스터, 월드, HUD를 모듈 단위로 유지한다.
- 매 프레임 새 객체를 과도하게 생성하지 않는다.
- 프로토타입에도 상태 전환과 수용 기준을 명시적으로 둔다.
