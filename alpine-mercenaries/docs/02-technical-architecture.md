# 기술 아키텍처

## 기술 결정

### 제작판

- 엔진: Unreal Engine 5.8.0 정식 버전
- 언어: C++ 중심, 디자이너 조정과 연출은 Blueprint 병행
- 렌더링: 사실적 3D, 3인칭 숄더뷰 카메라, Windows 우선
- 물리: 서버에서 재현 가능한 단순 충돌 프리미티브 우선
- 온라인: 1~5인 세션과 Steam 로비를 먼저 구현

사실적 캐릭터와 자연환경, 애니메이션, 물리 기반 부위 판정, PC 패키징과 Steam 연동 요구가 엔진 선택의 기준이다. 2026년 7월 23일 설치 가능한 정식 버전인 Unreal Engine 5.8.0으로 초기 프로젝트를 고정했다. 주요 마이너 업그레이드는 별도 브랜치에서 에셋·빌드·성능을 검증한 뒤 결정한다.

### 보관된 메커니즘 프로토타입

- Vite + Three.js
- 로컬 단일 플레이
- 원시 도형과 절차적 배치로 판정 가시화
- 최종 아트나 네트워크 구현으로 사용하지 않음
- 2026년 7월 23일 이후 기능 개발과 플레이 검증에 사용하지 않음

프로토타입은 설계 참고 자료로만 보존한다. 코드나 장면을 제작판으로 이식하지 않고, 문서화된 요구사항을 Unreal 프로젝트에서 처음부터 재구현한다.

## 런타임 구조

```text
Steam 인증/로비
      |
소셜 허브 서버 ─ 거래·채팅·파티·의뢰
      |
전투 인스턴스 서버(1~5인)
      |
계정·캐릭터·아이템·거래 영속 저장소
```

초기 온라인 알파는 한 파티원이 호스트가 되는 Steam 로비 기반 listen server로 비용을 낮출 수 있다. 거래와 아이템 소유권은 별도 권위 서비스가 결정해야 한다. 공개 서비스 단계에서는 치트와 호스트 이탈 대응을 위해 전투 전용 서버 전환을 평가한다.

## 서버 권위가 필요한 항목

- 피해, 회복, 상태이상과 사망
- HP·스태미너·조건부 MP의 소비와 회복
- 방패의 공격 차단과 뒤쪽 아군 보호
- 부위 피해, 파괴와 보스 상태 전환
- 드롭 생성, 아이템 고유 ID와 소유권
- 현재 상태, 최대 내구도, 수리와 불멸화
- 직접 거래, 거래 게시판 체결과 재화 이동
- 퀘스트 완료와 보상 지급

클라이언트는 입력과 표현을 담당하고 서버 결과를 예측해 부드럽게 보여준다. 아이템 수치와 거래 완료를 클라이언트 값만으로 결정하지 않는다.

## 주요 게임 모듈

- Character Locomotion: 걷기, 질주, 앉기, 점프, 넘기, 오르기
- Character Vitals: HP, 모든 물리 행동의 스태미너 소비·회복, 무기 조건부 MP
- Weapon Actions: 무기 입력, 콤보, 자원과 역할 행동
- Directional Defense: 방패 평면, 보호 볼륨, 완벽 방어와 쳐내기
- Target Anatomy: 몬스터 부위, 피해 분배, 파괴 후 행동 변경
- Encounter State: 보스 패턴, 예고, 목표, 실패와 완료
- Contract System: 의뢰 정의, 변형 조건, 파티 입장, 보상
- Item Authority: 고유 ID, 옵션, 내구도, 불멸화, 이력
- Trading: 직접 교환, 등록, 검색, 체결, 감사 로그
- Session: Steam 로비, 초대, 재접속, 호스트 이탈 정책

## 데이터 원칙

- 아이템 인스턴스마다 위조하기 어려운 서버 발급 고유 ID를 가진다.
- 소유권 이전은 단일 트랜잭션으로 처리하고 중복 소유를 허용하지 않는다.
- 거래창 변경 시 양쪽의 승인 상태를 초기화한다.
- 불멸화는 되돌릴 수 없는 아이템 상태지만 거래 제한은 만들지 않는다.
- 콘텐츠 정의와 플레이어 소유 데이터의 스키마를 분리한다.
- 라이브 밸런스 변경이 기존 아이템 인스턴스를 손상하지 않도록 버전을 기록한다.

## Steam 연동 방향

Unreal의 Steam 온라인 서브시스템은 로비, 매치메이킹, 업적, 음성, 클라우드 같은 인터페이스를 제공한다. 첫 Steam 통합 범위는 다음으로 제한한다.

1. Steam 사용자 인증
2. 친구 초대와 1~5인 로비
3. 로비에서 전투 인스턴스 진입
4. 업적
5. 클라우드 설정 저장
6. Windows 패키지의 SteamPipe 업로드

아이템 경제 데이터는 Steam 클라우드 파일이 아니라 권위 백엔드에 저장해야 한다.

## 개발 환경 필요 사항

- Unreal Engine과 Epic Games Launcher
- 충분한 SSD 여유 공간과 제작판용 소스 관리
- C++ 툴체인: macOS 개발 시 Xcode, Windows 빌드 시 Visual Studio Build Tools
- Git LFS 또는 대용량 아트 자산 저장소
- Blender 또는 동등한 DCC 도구
- 캐릭터 리깅·애니메이션 제작 또는 합법적으로 라이선스된 자산
- Steamworks 파트너 계정과 출시 시점의 App ID
- Windows 실기기 테스트 환경

현재 개발기는 Apple M4와 24GB 메모리를 사용한다. 2026년 7월 23일 Epic Games Launcher 로그인과 Unreal Engine 5.8.0 설치를 완료했고, 엔진은 `/Users/Shared/Epic Games/UE_5.8`에 있다. Git LFS 3.7.1도 프로젝트 저장소에 활성화했다. `/Applications/Xcode.app`에 Xcode 26.1.1을 설치했고 Metal 컴파일러와 macOS 26.1 SDK를 확인했다.

[Epic의 UE 5.8 macOS 개발 요구사항](https://dev.epicgames.com/documentation/en-us/unreal-engine/macos-development-requirements-for-unreal-engine)에 따르면 최소 Xcode는 16.4, 권장 버전은 26.1.1이다. Xcode 26.4는 호환되지 않는다. 저장소의 검증·빌드 스크립트는 `DEVELOPER_DIR`를 통해 설치된 26.1.1을 명시적으로 선택한다. 시스템 `xcode-select`는 여전히 Command Line Tools를 가리킬 수 있으므로 필요할 때 사용자가 관리자 권한으로 전체 Xcode 경로를 전환한다.

## 비기능 요구사항 초안

- 전투 인스턴스 목표 서버 틱: 측정 후 30Hz 이상 검토
- 온라인 지연 100ms에서 완벽 방어가 납득 가능하도록 입력 유예와 서버 보정
- 색상에만 의존하지 않는 공격 높이·방향 예고
- 키보드·마우스와 게임패드 기능 동등성
- 연결 해제 후 동일 의뢰로 재접속
- 모든 아이템 생성·이동·소비·복원·불멸화 감사 로그
- 개발, 스테이징, 라이브 경제 데이터 완전 분리
