# 개발 로드맵

## 공통 완료 조건

모든 단계는 기능 구현만으로 완료되지 않는다. 관련 테스트와 빌드가 통과하고, 해당 단계만 담은 커밋이 GitHub 원격 저장소에 푸시되어야 완료로 처리한다. 다른 프로젝트의 작업 중 변경은 단계 커밋에 포함하지 않는다.

- [x] 무기가 직업을 대신하는 역할 구조 확정
- [x] 방향 방패, 아군 보호, 쳐내기 핵심 규칙 확정
- [x] 걷기, 질주, 앉기, 점프, 파쿠르 요구 확정

- [x] 이중 내구도와 극희귀 불멸화 광물 확정

## 단계 0: 웹 메커니즘 샌드박스(종료·보관)

목표는 최종 그래픽이 아니라 조작과 판정의 재미를 확인하는 것이다.

- [x] 프로젝트 문서와 기술 방향 작성
- [x] 3인칭 숄더뷰 카메라, 마우스 회전과 WASD 이동
- [x] 질주, 앉기, 점프와 낮은 장애물 넘기
- [x] 용병 사무실과 의뢰판 상호작용
- [x] 알프스 테스트 전투 맵 전환
- [x] 검방 방향 방어와 보호 판정
- [x] 브레스 공격과 뒤쪽 아군 보호 피드백
- [x] 활 정밀 조준과 보스 약점
- [x] 당시 빌드와 브라우저 수동 검증
- [x] 브라우저 기반 기능 개발과 완료 검증 중단

이 단계는 제작판의 품질 또는 기능 완료를 증명하지 않는다. 이후 단계에서 브라우저 실행 결과를 수용 기준으로 사용하지 않는다.

완료 기준: 웹 서버 없이 Unreal Editor에서 3인칭 캐릭터가 정상 자세로 생성되고, WASD 이동·마우스 숄더 카메라·달리기·앉기·점프·기본 충돌이 검증된다.

공식 Third Person 템플릿의 프로젝트·맵·에셋 참조와 Mac Metal 네이티브 패키징을 검증했다. 패키지 실행 화면에서 캐릭터가 똑바로 서고 기본 3인칭 카메라가 동작하는 것도 확인했다. 템플릿의 기본 카메라와 이동을 게임 고유 수용 기준의 완료로 계산하지 않는다.

## 인력·자산 현실성 점검

최종 미술 목표는 프로그래밍만으로 달성되지 않는다. 최소한 다음 전문 작업이 필요하다.

- 캐릭터 모델링, 헤어, 의상과 장비
- 몬스터 콘셉트, 모델링, 리깅과 애니메이션
- 환경 아트, 식생, 조명과 VFX
- 전투·파쿠르 모션과 사운드
- 서버, 데이터베이스, 보안과 운영
- UI/UX, 접근성, QA와 현지화

초기에는 합법적으로 사용 가능한 임시 자산을 활용하되, 스토어 페이지에 최종 품질로 오해될 자료를 사용하지 않는다.

## 가장 큰 위험

1. 사실적 파쿠르와 정밀 전투 애니메이션의 제작량
2. 쿼터뷰에서 높이와 부위 조준을 읽는 문제
3. 지연 환경에서 완벽 방어가 불공정하게 느껴지는 문제
4. 자유 거래에서 아이템 복제, 봇과 현금 거래가 발생하는 문제
5. 불멸화 아이템이 누적되어 장기 제작·드롭 경제가 정체되는 문제
6. 한 허브와 반복 의뢰가 콘텐츠 부족으로 느껴지는 문제

각 단계는 다음 단계의 대규모 제작 전에 해당 위험을 실제 플레이로 검증한다.
